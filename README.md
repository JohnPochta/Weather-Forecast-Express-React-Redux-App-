Раздел 1. Аннотация.
Данный web-service был создан исходя из задания поставленного перед разработчиком:

Сделать одностраничное приложение для отслеживания погоды(апи брать какие будет удобно)
Приложение должно уметь:
1.Добавлять/удалять города
2.Запрашивать данные для выбранного города на данный момент + ближайший прогноз
3.Автоматически запрашивать погоду по координатам пользователя — это город/место по умолчанию.
4.Покрытие кода комментариями не менее 40%
5.Данные не должны пропадать после обновления страницы
6.Результат работы предоставить в гит репозитории

Обязательно:
Работу вести в гит репозитории
Минимальная документация (readme.md)

Будет плюсом:
Использование предпроцессоров
Покрытие кода автоматическими тестами
Использование css фреймворков

Важно:
Использование create-react-app является минусом

Задание было выполнено с использованием следующих технологий:
1. Серверная часть (back-end) приложения - фреймворк express (из пакетной системы node.js)
2. Клиентская часть (front-end) - библиотека React JS + технология Redux. Исходя из поставленных условий для сборки react app использовался webpack, а не create-react-app
3. Пользовательские данные хранятся в нереляционной БД - Mondo DB, работа с которой в данном проекте реализуется с использованием ORM Mongoose.
4. В качестве замены css фреймворкам было использовано фреймворк для создания переносимых интерфейсов React-Semantic-UI.
P.S.: Redux был использован , поскольку в данный проект было принято решение интегрировать уже готовый авторский шаблон для авторизации/аутентификации на клиентской части.
Сам по себе шаблон(речь идёт о компоненте Start Page и всех привязанных к нему reducer-ов из папки reducers) реализован давно и не очень аккуратно, но может быть переделан=)
Сама же технология Redux, кроме этого шаблона больше в проекте нигде не используется по ненадобности. 
Хотя, есть места, где её можно было бы применить, но это уже совсем другая история.

В front2/components - расположены компоненты
в front2/containers - контейнеры, в понимании их сущности, описанном выше).

Функционал приложения:
1. После прочтения ТЗ к приложению - стало очевидно, что необходимо реализовать авторизацию и аутентификацию, поскольку у каждого пользователя должен быть сформирован его личный список избранных городов.
Для реализациии данной задачи использовались средства Node.js, позволяющие реализовывать крипто-защиту паролей, cookies и т.д. Это всё сказано лишь для того, чтобы было понятно, что данные надёжно защищены).
2. Данное приложение составлено из двух страниц: начальной(регистрация+авторизация) и главной(просмотр прогогноза погоды).
3. На главной странице реализовано просмотр погоды по текущей геолокации, с помощью нативных js средств: navigator. ВОт только данная функция не всегда работает=).
Эта тема освещена здесь: https://stackoverflow.com/questions/3397585/navigator-geolocation-getcurrentposition-sometimes-works-sometimes-doesnt . Так что, прогноз погоды по заданной геолокации иногда будет присутвовать - иногда появлятся спустя время. Это никак не связано с ошибками в коде, поверьте).
4). Стоит упомянуть, что в данном приложении реализовано просмотр прогноза погоды в городах Украины, поскольку такое решение наиболее оптимально для демонстрации(что является целью теста) навыков работы с api. Кстати, используемое API: https://openweathermap.org.
Почему только Украина? Потому что нам нужно каким-то образом идентифицировать города. В идеале , это конечно же, лучше делать по PrimaryKey: city_id, но тогда нам нужно иметь датасет в котором есть чёткоет соотвествие : город - id. 
В данном api городов около миллиона, а, следовательно, обработка таких данных - это уже, Big DATA, поэтому было выбрано простой способ идентификации города, за которым пользователь хочет следить:
Когда пользователь вводит город - посылается запрос на Api по city_name и указанием страны(ua), и в случае, если запрос состоялся (не failed ), то значит такой город можно внести в свой список и об этом пользователь моментально осведомляется).
5. Для выбраных городов можно смотреть прогноз погоды и удалять их). 
6. При добавлении нового города, чтобы обеспечить появление его в списке городов, во избежание использования redux - используется кнопка refresh CitiesList=).
7.
